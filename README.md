# Safety Gate Landing Page

Static landing page for **Safety Gate** — approval-aware MCP security middleware for local AI agents.

## What this site is for

This landing page positions Safety Gate as:

- a policy and approval layer for MCP tool execution
- a local-first control layer for shell and file access
- a middle ground between blind trust and full sandboxing

It is meant to explain, quickly:

- what Safety Gate is
- what problem it solves
- how it differs from orchestration and isolation tools
- why a security-conscious AI team would care

## Current messaging focus

The site now emphasizes:

- policy-enforced tool execution
- approval workflows for risky actions
- approver / executor identity
- approval expiry and replay protection
- auditability for shell and file tool execution
- OpenClaw / MCP-friendly positioning
